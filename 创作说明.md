# Ancient Arch Wonders 创作说明

## 一、作品概述

**Ancient Arch Wonders**（中文名：古建筑奇境）是一款面向 Windows 的 **2D 综合互动作品**：以角色在院落、桥面的行走与剧情为骨架，穿插榫卯配对、卷轴交互、平台跳跃、季节罗盘与修缮等模块，并以分镜与视频完成叙事收束。对外名称、可执行文件名与封面视觉保持一致即可。场景美术参考传统院落空间，内容侧重**古建筑修缮中的通用知识**，而非特定景点推广。

## 二、创作动机

教材中对榫卯、构件配合、热胀冷缩等多以定义呈现，记忆成本高。本作品尝试将抽象表述转化为**可重复的操作**：拖放对准、测量到位后再确认、罗盘对应季节、险情下的修复步骤等，使玩家在动手过程中建立直观印象。叙事上希望强调**技艺的传承**与**修缮中的审慎态度**，结尾以静图与视频收束情绪与主题。

## 三、体验结构

整体链路为：主菜单进入开场（静图、序列帧、短视频与黑场组合；部分节点需玩家点击推进）→ 第一章对话 → 户外至门口交互 → 规则视频 → 榫卯关（三槽完成）→ 第一章收束视频 → 第二章（标题、进门、卷轴、横版平台关、通关视频及小知识页）→ 第三章卢沟桥（行走、对话、进入罗盘与季节相关玩法）→ **GameEnding** 多段图与视频 → 返回主菜单。

关卡跳转由各场景内配置的 **nextSceneName** 等逻辑驱动，与 Build Settings 中的场景列表顺序**不必一一对应**；评审时以当前构建版本实际流程为准。

## 四、玩法模块说明

- **榫卯**：拖入正确卯口 → 微调对齐 → 测量指针进入绿色区后 **Enter** 完成「彻底铆合」；中间接入测量条，表达「到位前仍需一步确认」。为防止跳过教学顺序，**未完成当前槽铆合时**不可切换至其他槽，并以提示反馈。  
- **户外与门区**：承担空间过渡，由「外部观察」进入「室内规则学习」。  
- **卷轴**：以卷轴为容器，配合飞入、点击展开与图文揭示，串联信息呈现。  
- **平台关**：收集场景构件、躲避障碍、在出口区域内确认离开；体量控制为短关卡，避免节奏拖沓。  
- **罗盘**：点击转盘，指针停落扇区对应季节，视频循环播放；夏 / 冬等节点进入**修复模式**，以方向键微调至安全区间并配合倒计时；失败可重试。  
- **结尾段**：以黑场、视频与插图交替，按短片终章节奏编排，最后回到主菜单。

## 五、视听与界面

整体追求**信息可读、控件易点**：色调偏木色与灰调，避免与高古建气质冲突。角色采用待机图与行走序列帧切换，未使用骨骼动画。字幕区域保留底条，保证夜间或暗场视频下的可读性。音乐与提示音分层处理，避免掩蔽关键反馈；罗盘处循环氛围声与短促「咔嗒」类音效分轨调音量，便于混音平衡。

## 六、技术实现要点

- **引擎**：Unity / 团结引擎，具体版本见工程 `ProjectVersion`。  
- **UI**：uGUI。  
- **过场视频**：`VideoPlayer` 渲染至 `RenderTexture`，再贴于 `RawImage`。开发中曾遇父节点未激活时出现「有声无画」等问题，衔接段对父节点激活状态有专门处理。  
- **场景切换**：`SceneManager`；黑场过渡采用全屏 `Image` 控制透明度。  
- **分辨率**：内容以 **1920×1080** 为基准制作；辅以脚本在宽、窄屏下裁黑边并统一画布缩放，降低答辩机分辨率差异带来的版式漂移。

实现上较费工的部分包括：开场与结尾多段视频与对白的时间对齐；罗盘角度在跨越 0°/360° 时的区间判定；平台关收集数量与出口判定的逻辑完备性，避免误判为程序故障。脚本命名、资源排布与排错说明见仓库内 **游戏设计文档** 与 **`Assets/README.md`**，本说明不重复罗列。
