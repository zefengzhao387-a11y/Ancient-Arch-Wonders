# Ancient Arch Wonders（古建筑奇境）— 游戏设计文档（详版）

> **文档定位**：面向评审、课程答辩与团队交接，说明设计目标、流程、交互、技术与美术取向。  
> **与工程的关系**：具体参数、拖引用路径、菜单工具入口等运维细节见 **`Assets/README.md`**；包依赖版本见 **`Packages/manifest.json`** / **`packages-lock.json`** 或发布目录中的《依赖库说明》。

## 一、游戏简介

### 1.1 作品概述

《**Ancient Arch Wonders**》（中文可称「**古建筑奇境**」；工程内素材说明亦沿用「乔家大院」叙事语境）是一款运行在 **Windows PC** 上的 **2D 综合互动作品**。它以**中国传统古建筑及其修缮常识**为知识载体，将**叙事分镜、轻解谜、横版探索、小游戏与视频演出**串联为一条可通关的体验链路。

玩家以角色「**乔乔**」的视角，在院落与桥面上移动，完成**榫卯配对**、**卷轴互动**、**平台收集跳跃**、**罗盘式季节—伸缩缝**等玩法，并在章节之间通过**黑场、淡入淡出、配音与字幕**完成情绪与信息的递进，最终在**结尾多段视频与插图演出**中收束主题。

### 1.2 类型与体验特征

| 维度 | 说明 |
|------|------|
| 类型标签 | 2D、叙事驱动、关卡化小游戏合集、文化科普向 |
| 操作重心 | 键盘移动/跳跃/交互 + 鼠标点击与拖拽（因关卡而异） |
| 单局体量 | 以「短关卡 + 强反馈」为主，便于课堂演示与反复试玩 |
| 技术形态 | Unity / **团结引擎** 发行版；以 **uGUI** 搭建界面，**VideoPlayer + RenderTexture** 承载过场与环景视频 |

### 1.3 目标用户与使用场景

- **课程 / 竞赛评审**：可在 10～20 分钟内走完主线，亮点分散在「文化符号玩法化」与「演出管线自研」两处。  
- **一般玩家**：无需查阅文档即可依赖 UI 提示与规则视频上手；失败（如时间到、平台坠落）提供重开或回到安全节点。  
- **维护者**：策划可在 Inspector 中替换对话、图片、音视频与部分时长参数，程序员主要维护控制器状态机与边界情况。

### 1.4 体验节奏（叙事顺序简述）

**主菜单** → **开场分镜**（多页图文/动画/视频）→ **第一章前置对话** → **户外行走**（门前对话）→ **规则视频** → **榫卯小游戏**（含测量桥接）→ **第一章结束过渡**（视频 + 自动对白）→ **第二章**（标题 / 对话 / 竖井或卷轴 / 平台关 / 通关视频与小知识）→ **第三章标题** → **卢沟桥**（行走 + 罗盘 + 知识补充 + 通关后短片）→ **结尾演出** → **回主菜单**。

> 注：实际跳转以各场景 **`ChapterTransitionController` / 各 *Controller` 的 `nextSceneName` 与按钮回调** 为准；`EditorBuildSettings` 中登记了全部可构建场景，顺序不一定等于游玩顺序。

## 二、创作立意

### 2.1 主题与价值观

1. **可玩的建筑语言**  
   建筑术语若仅出现在字幕里，记忆留存率低。本作尝试把「**契合**」「**零位/容许误差**」「**季节变化对构造缝的影响**」等概念，转译为**拖拽匹配、指针读数、罗盘扇区与修复操作**，使玩家在「做对动作」时自然完成知识内化。

2. **传承与责任的叙事框架**  
   开场分镜强调代际与技艺传承；结尾回到现实与「木盒」意象，形成**情感闭环**。技术实现上依赖**可配置的步骤序列**与**多段视频**，便于替换文案而不改代码结构。

3. **敬畏与修缮伦理**  
   通过「危险季节需修复」「修复成功才解除风险」等机制，传达**预防性保护**与**干预需谨慎**的态度，而非鼓励无依据的「大拆大建」。

### 2.2 教育与娱乐的平衡

| 策略 | 说明 |
|------|------|
| 短关卡 | 单机制关卡时间可控，适合答辩限时演示。 |
| 规则显性化 | 规则视频 + 弹窗说明，降低认知门槛。 |
| 知识卡片化 | 第二章末、第三章末等节点插入**小知识插图 + 继续**，不打断大节奏。 |
| 可重复性 | 时间类失败（如罗盘倒计时）支持重开整局小游戏，减少挫败感。 |

### 2.3 与「乔家大院」语境的关系

工程 `Assets/README.md` 以「乔家大院」为场景叙事锚点（院落、户外门区、榫卯工序等）。产品对外名称可为 **Ancient Arch Wonders**，在答辩材料中建议**统一封面名与 exe 产品名**，并在文档首段用一句话说明「叙事场景以晋商院落与卢沟桥等为舞台，知识指向通用古建修缮概念」。

## 三、核心玩法（分模块详述）

### 3.1 主菜单（`GameMenu`）

- **内容**：全屏或大幅背景图、「游戏开始」「退出游戏」等按钮；可挂持久 BGM。  
- **设计意图**：建立视觉基调；`nextSceneName` 可配置为进入开场或调试入口。  
- **技术要点**：`GameMenuController`；与 `PersistentGameBGM`、`GameUISfxHub` 协同注意**不要把需 DontDestroyOnLoad 的组件挂在整块 Canvas 根上**（详见脚本注释），避免场景切换后 UI 丢失。

### 3.2 开场分镜（`OpeningVideo`）

- **步骤类型**（`OpeningVideoController`）：`Image`（静图）、`FrameAnimation`（序列帧）、`Video`（短视频）、`BlackTransition`（黑场过渡）；已弃用的 `TravelEffect` 若仍留在序列中会被跳过并报警，应删除以免黑屏无内容。  
- **单步能力**：每步可配 `text` / `voice` / `minDuration` / `extraDelay` / `waitForClick`；亦支持**同一步内多句字幕**（`subtitleBeats`），用于同一画面内的多句对白。  
- **视频步进阶**：  
  - **顺序模式**：各 beat 的 `showAtTimeInVideo` 全为 0 时，按列表顺序随播放切换字幕与配音；  
  - **时间轴模式**：`showAtTimeInVideo` 有正值时，按视频时间触发对应句。  
- **章节衔接**：全部步骤结束后 → 黑场 → **衔接视频**（`bridgeTransitionClip` 或 `StreamingAssets` 下文件名）→ 再黑场 → 加载 `nextSceneName`（默认 `Chapter1Intro`）。若未配置衔接视频，则仅最后黑场后进入下一章。

### 3.3 第一章前置对话（`Chapter1Intro`）

- **流程**：黑场渐显全屏背景 → 延迟后渐显第一张对话框图 → 点击继续 → 渐隐再渐显第二张 → 再点击进入户外。  
- **可选配音**：`firstDialogVoice` / `secondDialogVoice`。  
- **技术要点**：`Chapter1IntroDialogController`；背景与对话框为**两层 Image**，便于美术把字画在图上或后续改为 TMP 叠字。

### 3.4 户外探索（`Outdoor`）

- **操作**：`QiaoQiaoPlayerController` — **A/D**（`Horizontal` 轴）横向移动，多帧行走与待机图切换；移动后隐藏「按 AD 移动」提示。  
- **门区交互**：进入触发带后，可按 **E** 进入门内流程；支持进门前 **1～2 张叠层对话图**（第二张 Sprite 不配则第一张继续后直接进入下一场景）。  
- **叙事意图**：从「院落外」进入「规则与正式玩法」的心理过渡。  
- **技术要点**：`OutdoorSceneController`；门区用 Rect 重叠判定；配音可走独立 `AudioSource`。

### 3.5 规则视频（`RulesVideo`）

- **内容**：播放规则说明视频，按钮进入榫卯关或下一环节。  
- **设计意图**：统一玩家对「本关要做什么」的认知，减少误操作投诉。  
- **技术要点**：`RulesVideoController`（与工程内命名一致即可）。

### 3.6 榫卯小游戏（`TenonMortiseGame`）

- **核心操作**：鼠标将**可拖榫件**拖入上方**卯槽**（`DropZone`）。  
- **阶段反馈**：空槽 → **微微铆合**（替换槽位图，可 `useCustomRect` 精细对齐）→ **完全铆合**（可触发介绍图 `Intro Sprite`）；与 **测量条**（`MeasurementBarController`）联动：指针归零容差、确认键（默认 **Enter**，可改为避免与编辑器冲突的键）。  
- **设计意图**：把「配合公差」「逐步到位」转化为可视、可听的渐进反馈。  
- **技术要点**：`DraggableItem`、`DropZone`、`MatchReaction` / `MatchToMeasurementBridge`、`TenonMortiseIntroDisplay`；场景进入常挂 **`SceneFadeIn`** 做渐显。

### 3.7 第一章结束过渡（`Chapter1Outro`）

- **内容**：短视频播完可停在最后一帧；两段 UI Text 对白渐显，可配 `dialogVoice1/2`。视频来源支持 Clip 或 `StreamingAssets/chapter1_outro.mp4`。  
- **设计意图**：第一章情绪收束，并**无操作负担**地接入第二章。

### 3.8 第二章（多场景组合）

典型包含（以 Build Settings 中启用场景为准，顺序由跳转逻辑决定）：

| 场景（示例） | 玩法摘要 | 代表脚本 / 控制器 |
|--------------|-----------|-------------------|
| `Chapter2Title` | 章节标题卡 | `Chapter2TitleController` 等 |
| `Chapter2Intro` | 门前对话、进门 | `Chapter2IntroController` |
| `Chapter2Shuijing` | 竖井 / 特定叙事关 | 对应场景控制器 |
| `Chapter2Scroll` | 卷轴拖拽浏览 | `Chapter2ScrollController`、`ScrollRibbonDrag` |
| `Chapter2Platformer` | 横版跳跃、收集物、尖刺、出口 | `Chapter2PlatformerController` |
| `Chapter2VideoEnd` | 通关视频 + 自动字幕 + 小知识图 | `Chapter2VideoEndController` |

- **平台关要点**：**W / 空格 / ↑** 跳跃；收集够数且角色与出口区重叠时提示按 **S** 离开；坠落触发游戏结束面板与重开。  
- **设计意图**：在第二章内完成**玩法形态切换**，避免玩家对「只会点击」产生疲劳。

### 3.9 第三章标题与卢沟桥（`Chapter3Title`、`Chapter3Bridge`）

- **桥面阶段**：类平台移动、跳跃、与桥面碰撞；至中心区触发对话链；按 **E** 进入玩法介绍再进入罗盘界面。  
- **罗盘阶段**：  
  - 点击盘面后指针做**带圈数的缓动旋转**，停于随机角度；根据指针所在**角度扇区**判定春/夏/秋/冬，并在对应格播放 **循环季节视频**（`VideoPlayer` + `RawImage` + `RenderTexture`）。  
  - **夏季 / 冬季**进入「危险—修复」叙事：进入**修复模式**，按住 **↑ 或 ↓** 微调指针至安全文案所述区间；成功则计数，失败条件含**倒计时结束**（可弹窗重开罗盘局）。  
  - **音效分层**：自动旋转循环音、落定音、指针对准季节成功音、键盘交互音等分字段配置音量。  
- **通关后**：可播放 `postWin` 视频（Clip 或 StreamingAssets），再接对话框与知识面板。  
- **技术要点**：`Chapter3BridgeController`、`CompassSeasonVideoGlow`；视频 `Prepare` 异步与 UI 显隐需防止「只闻其声不见其画」。

### 3.10 结尾演出（`GameEnding`）

- **结构**（`GameEndingController`）：黑场渐显 → 首图 + 首段字幕（可配音，最短停留与音频长度取大）→ 点击继续 → **视频 1** 与**两段**下方字幕（可与视频并行至结束）→ 黑场过渡 → **视频 2** 与**一段**字幕 → 中间插图（点击）→ 终图 + 可选字幕（点击）→ 回 `GameMenu`。  
- **资源约定**：视频除拖 Clip 外，可放置 `StreamingAssets/ending_part1.mp4`、`ending_part2.mp4`。  
- **设计意图**：情感与信息密度最高的收束段，用**强节奏剪辑感**（黑场、双视频、插图）模仿短片终章。

## 四、交互机制（详述）

### 4.1 输入体系总览

| 输入 | 典型绑定 | 出现场景 |
|------|-----------|-----------|
| 横向 | `Horizontal`（**A/D**，部分系统含方向键） | 户外乔乔、第二章平台、第三章桥面 |
| 跳跃 | **W**、**空格**；平台关另支持 **↑** | 第二章平台、第三章桥面 |
| 交互 | **E** | 户外进门、第三章进入介绍/流程 |
| 出口确认 | **S** | 第二章平台关（需满足收集与站位） |
| 罗盘修复 | **↑ / ↓** 长按 | 第三章罗盘修复模式 |
| 鼠标左键 | UI Button、分镜继续、拖拽 | 全局 |
| 确认测量 | 默认 **Enter**（可改） | 榫卯测量条 |

### 4.2 UI 与叙事交互范式

1. **「黑场遮罩 + 插值 alpha」**：统一由 `Image` 全屏黑或 `fadeOverlay` 控制，避免场景底色素材不一致时露白。  
2. **「点击继续」透明按钮**：置于最上层或次上层，结束时 `SetAsLastSibling` 避免被遮罩挡住。  
3. **字幕条与底栏**：`SubtitleStyleUtility` 统一渐变条、衬线颜色与文字色，保证在视频暗部可读。  
4. **分镜与视频的时序耦合**：见第六节「叙事与媒体同步」——避免用「视频是否结束」打断尚未播完的多句配音。

### 4.3 音频交互范式

- **配音轨**：长对白使用 `AudioSource.clip` + `Play()`，便于按句 `Stop` 与算时长。  
- **UI 点击**：`GameUISfxHub` 单例 + `PlayOneShot`，避免每个按钮单独挂源。  
- **BGM**：`PersistentGameBGM` 跨场景时注意与 Sfx Hub **不要错误共用同一 AudioSource 并改写 loop**（脚本内已有注释约束）。  
- **罗盘**：旋转用**独立循环 AudioSource**，落定与成功用 **PlayOneShot** 或单次播放，音量字段拆分，避免「循环音量被 Clamp 到 1 无法调响」与「一次性音效混用同一缩放」的混淆。

### 4.4 状态机与防重入

- 大章节脚本（如 `Chapter3BridgeController`）采用 **Phase 枚举** 划分Walking / Dialog / Compass / PostWin 等，**同一帧内禁止重复进协程入口**；指针动画中用 `_pointerAnimating` 等标志防抖。  
- 场景切换：尽量在淡出结束后再 `LoadScene`，避免音频与视频在卸载时残留回调。

## 五、技术路线（详述）

### 5.1 引擎与版本

- **编辑器**：`ProjectSettings/ProjectVersion.txt` 记录为 **Unity 2022.3.62t3**，团结编辑器 **1.8.1**。答辩与提交材料应与此一致，避免「本地打不开工程」。

### 5.2 目录与职责（逻辑视图）

| 路径 | 职责 |
|------|------|

### 5.3 渲染与 UI 技术栈

- **UI**：`Canvas` + `Canvas Scaler`（Scale With Screen Size，参考分辨率多与 1920×1080 相关）+ `GraphicRaycaster`。  
- **视频**：`VideoPlayer.renderMode = RenderTexture`，贴到 `RawImage`；注意 **StoryPanel / 父节点激活状态**，否则会出现脚本注释所述「有声无画」。  
- **适配**：除 Scaler 外，工程内另有 **`Aspect16x9StrictEnforcer`**、**`GlobalCanvasAdaptation`** 等辅助脚本，用于在异形分辨率下尽量贴近设计画幅（以 Inspector 实际启用为准）。

### 5.4 数据驱动与配置边界

- **强数据驱动**：开场步骤数组、对话列表、视频与 Streaming 文件名、各阶段淡入淡出时长等多以 `[SerializeField]` 暴露。  
- **弱数据驱动**：复杂几何仍依赖场景内 RectTransform 布局；大改 UI 结构仍需在编辑器内操作。

### 5.5 依赖与第三方边界

- **包管理**：以 Unity / 团结 **官方 Registry** 为主（`com.unity.ugui`、`textmeshpro`、`timeline`、`feature.2d` 等），详见《依赖库说明》。

## 六、重难点说明（详述）

### 6.1 叙事与媒体同步（开场 / 规则 / 结尾）

**问题现象**：视频时长短于多句配音总时长时，若等待循环中误用「视频已结束」标志提前 `break`，会导致**仅首句或前半段有声、后续句被整段跳过**；或视频已停而字幕仍快速翻页。  

**设计—实现对策**：

1. **顺序字幕模式**：每一句按 `max(minHold, voice.length)` 等待完毕，**不**与 `_videoClipStepEnded` 做「与」短路。  
2. **时间轴字幕模式**：视频结束后若仍有未触发 cue，补播时**每句仍要等待完整 beat 时长**，不能「每帧播一句」。  
3. **衔接视频**：`PlayBridgeTransitionThenLoad` 中保持承载 `RawImage` 的父节点激活，避免黑场下视频纹理不渲染。  

### 6.2 分辨率、安全区与构建一致性

**问题现象**：编辑器 Game 视图比例与 Windows 播放器不一致时，UI 锚点与 Scaler 组合导致**裁切、贴边错位或「展示不全」**。  

**对策**：

1. 以 **1920×1080 16:9** 为设计基准，在说明文档与答辩中明确告知评委。  
2. Player Settings 中 **默认分辨率、全屏/窗口模式** 与答辩机一致；必要时用 16:9 约束脚本。  
3. 构建前用 **Standalone Development Build** 在目标分辨率下完整跑一遍全流程录屏。

### 6.3 榫卯—测量—介绍链路与协程挂载点

**问题现象**：`TenonMortiseIntroDisplay` 若挂在默认未激活的 Panel 上，协程无法启动，控制台报 *Coroutine on inactive object*。  

**对策**（README 已载）：将 Intro 控制器挂到**常驻激活**空物体，由 `MatchToMeasurementBridge` 引用；Intro 面板仅作视觉子节点。

### 6.4 第二章平台关：收集、出口与物理手感

**难点**：收集数、出口区重叠、尖刺判定、**S 键确认离开**需同时满足，否则玩家误以为 Bug。  

**对策**：出口区激活「按 S」提示 Image；仅在 `_collected >= totalToCollect` 且与 `exitZone` 重叠时监听 `S`；收集与掉坑分别走独立 UI 与音效反馈。

### 6.5 第三章：罗盘角度区间与视频性能

**难点**：  
- 角度在 0～360 环绕时，**跨 0° 的区间**（如夏 270～360）需单独处理 `from > to` 情况。  
- 四路视频同时存在时，**Prepare/Play** 与内存占用需控制，避免低配机卡顿。  

**对策**：统一用 `Mathf.Repeat` 与封装函数 `InAngleRange`；非当前季节视频及时 `Stop` 与隐藏；指针旋转用平滑插值（smoothstep）避免机械感。

### 6.6 音频：循环与一次性音效的增益模型

**难点**：`AudioSource.volume` 在引擎侧常见上限为 1，而 `PlayOneShot` 第二参数可大于 1；混用同一 `compassSfxVolume` 易产生「以为调了 1.5 但循环仍只有 1」的错觉。  

**对策**：拆字段 **`compassSpinLoopVolume`** 与 **`compassAutoSpinSettleVolume`** 等，分别接循环源与 OneShot。

### 6.7 工程体积、版本与提交

- **不提交** `Library`、`Temp`、`Logs` 等；大视频放 StreamingAssets 时控制码率。  
- **版本冻结**：答辩前锁定团结版本与包版本，避免锁文件漂移导致合并冲突。

## 七、美术风格说明（详述）

### 7.1 整体视觉方向

- **世界观可读性优先**：2D 横版与全屏 UI 混排，**高对比前景 + 适度压暗背景**，保证玩法区与对话区一眼可辨。  
- **文化感**：色彩上倾向**木色、青砖灰、淡金点缀**，避免高饱和荧光色干扰古建气质（具体以实配资源为准）。  
- **统一字幕语言**：字幕条 + 浅色正文的组合，保证在视频夜景段落仍可扫读。

### 7.2 角色与动画

- **乔乔**：待机单图 + 行走循环帧；通过 `localScale.x` 翻转朝向；脚步音可按行走帧节拍播放并设**最小间隔**防密。  
- **表情与剧情**：重要情绪多放在**立绘或全屏插图**上，减少骨骼动画依赖，降低制作成本。

### 7.3 UI 与信息层级

- **一级信息**：当前任务目标、倒计时、操作提示（如「按 S 离开」）。  
- **二级信息**：知识补充、历史旁白。  
- **控件尺寸**：按钮热区不小于推荐点击区域；重要按钮避免挤在角落以防误触。

### 7.4 视频与静图衔接

- 视频渐显时常与**黑场同步插值**，避免第一帧前露底。  
- 视频停于最后一帧时，注意与**后续自动对白**的时间轴对齐，避免闪一下黑。

### 7.5 音效与音乐

- **BGM**：音量偏低、循环平滑；不要与 UI 强提示音抢同一频率段。  
- **环境音**：院落、桥面可走环境音（若资源允许）增强沉浸。  
- **罗盘**：机械感循环 + 短促落定音，强化「机关感」。
